## Важные расхождения desktop vs mobile

- Mobile — это не просто узкая версия desktop; там изменены:
  - некоторые описания секций;
  - CTA-подписи кнопок на карточках;
  - текст в нижней CTA;
  - текст в final section;
  - поведение search/input в CTA.
- Desktop CTA поле показывает значение `КОНТЕНТ 2020–2024`.
- Mobile CTA поле показывает плейсхолдер `Введите надпись`.
- Mobile feature descriptions в Figma используют другой tone of voice и другой набор брендов/примеров.

## Что ограничило точность извлечения

- В текущем `figma_desktop` MCP нет rich-инструментов уровня `get_design_context` / `get_metadata(nodeId=...)` / `get_screenshot`.
- Канал `r1csa9n3` дал доступ только к:
  - document info
  - selection
  - pages
  - styles
  - variables
  - local components
  - PNG export
- Попытка получить SVG-экспорт оборвала соединение плагина.
- Повторное подключение к каналу после этого уже не прошло.

## Практический вывод для реализации 1:1

У нас уже есть достаточно данных, чтобы:

- воспроизвести порядок и состав секций;
- повторить desktop/mobile композицию;
- использовать те же Figma asset URLs, которые уже зашиты в текущем `index.html`;
- выверять типографику, размеры и отступы по PNG-референсам.

Но для совсем буквального 1:1 на уровне всех текстовых слоев и точных numeric values желательно еще один живой проход по Figma с рабочим каналом или более полным MCP-сервером.
